HTML списки

Списки предназначены для отображения контента объединенного общим смысловым значением.
Например: список контактов, список услуг, список товаров, список пунктов навигации, список изображений и т.д.

Упорядоченный список (ordered list)

Тег <ol> служит для отображения упорядоченного (нумерованного) списка.
Тег <ol> содержит в себе теги <li> - пункты списка (list items). 

            <ol>
              <li>list item 1</li>
              <li>list item 2</li>
              <li>list item 3</li>
            <ol>
  1. list item 1
  2. list item 2
  3. list item 3

Неупорядоченный список (unordered list)

Тег <ul> служит для отображения неупорядоченного (маркированного) списка.
Тег <ul> содержит в себе теги <li> - пункты списка (list items). 

            <ul>
              <li>list item</li>
              <li>list item</li>
              <li>list item</li>
            <ul>
  • list item
  • list item
  • list item

Таблицы

Для добавления таблицы на страницу используется тег <table>
Как и любая таблица, html-таблица состоит из строк и ячеек.
Для строк используется тег <tr>
Для ячеек используется тег <td>
Тег <th> определяет ячейку заголовка таблицы (table header cell )

Структура html-таблицы

      <table>
        <caption>Заголовок таблицы</caption>
        <thead><!-- шапка таблицы (table head) --></thead>
        <tbody><!-- тело таблицы (table body) --></tbody>
        <tfoot><!-- подвал таблицы (table foot) --></tfoot>
      </table>

Объединение ячеек

Для объединение ячеек существуют специальные атрибуты.
rowspan Для объединение ячеек по вертикали
colspan Для объединение ячеек по горизонтали
Пример:
<td rowspan='2' colspan='2'></td>

Изображения

Чтобы добавить изображение на страницу используется тег <img>
Есть два обязательных атрибута для тега <img>
<src> содержит путь к файлу изображения
<alt> содержит описание изображения
Пример:
<img src='cat.png' alt='котик'>

Относительные и абсолютные пути

Относительный путь - это путь по отношению к текущей директории. '../images/cat.png'

../ - подняться на один уровень ../../ - подняться на два уровня

Абсолютный путь - это путь от корневого каталога. '/images/cat.png' 'https://sitename.com/images/cat.png'

Форматы изображений

Универсально формата изображений на данный момент не существует. Есть большое количество различных форматов, каждый из которых обладает своими преимуществами и недостатками.

Vector vs Raster

SVG (Scalable Vector Graphics)

SVG - это язык разметки масштабируемой векторной графики. SVG графика строится на основе таких примитивов как: линия, окружность, элипс, путь (path).
Преимущества: Качество не страдает при масштабировании. Небольшой вес. Есть возможность взаимодействия через CSS и JavaScript (анимации).
Недостатки: Ограниченная область применения (как правило небольшие иконки, интерфейсная графика).
svg file

jpeg (jpg)

Формат jpeg является одним из наиболее популярных форматов графики. Преимущества: относительно небольшой вес. Высокая поддержка браузерами.
Недостатки: не поддерживает прозрачность (альфа канал).

png

Наряду с jpeg является одним из наиболее популярных форматов графики, но в отличии от него поддерживает прозрачность. Преимущества: Поддерживает прозрачность. Высокая поддержка браузерами.
Недостатки: вес файла.
Задача: добавить в html документ png изображение

gif

Используется для добавления анимированной графики.
Рекомендуется избегать использования данного формата. Вместо него стоит использовать видео-форматы (mp4, webm).

webp | webp

webp - это не один а целых два формата (сжатие с потерями и без потерь качества).
Преимущества: Небольшой вес. Поддержка прозрачности.
Недостатки: Есть проблемы с поддержкой. Необходимо добавлять fallback.
