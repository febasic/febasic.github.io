Инструменты для тестирования

  • https://web.dev/measure/

Важно

Не стоит слепо внедрять все рекомендации Google PageSpeed. Большая часть этих рекомендаций - это гипотезы, которые нуждаются в тестировании.

Метрики

  • FCP (First Contentful Paint)
  • SI (Speed Index)
  • LCP (Largest Contentful Paint)
  • TTI (Time to Interactive)
  • TBT (Total Blocking Time)
  • CLS (Cumulative Layout Shift)

Critical CSS*

Critical CSS - это практика размещения стилей, необходимых для отрисовки первого экрана сайта, непосредственно внутри head с помощью тега style.
Подробнее: Extract critical CSS
Стоит предварительно обсудить с командой. Внедрение Critical CSS может привести к ухудшению поддержки проекта.

Оптимизация шрифтов

  1. Используйте формат woff2 (+ woff, как fallback для IE11)
  2. Используйте font-display: swap
  3. Правильно подбирайте fallback для кастомного шрифта
  4. Удаляйте ненужные глифы из шрифта

Lazy Loading

  • loading='lazy' для изображений
  • loading='lazy' для iframe'ов
  • preload='none' для video

Не используйте формат gif

Вместо формата gif стоит использовать видео-форматы (mp4, webm). Необходимые атрибуты для тега video: autoplay, loop, muted, playsinline*.
Подробнее: Use video formats for animated content
* атрибут playsinline необходим для корректного воспроизведения видео на iOS устройствах. Подробнее: New video Policies for iOS

Подключайте внешние библиотеки правильно

Стоит подключать только необходимые модули из внешних библиотек (к примеру bootstrap). Это позволит минимизировать вес скриптов и стилей.

Другие рекомендации

  • Не используйте preloader
  • Не используйте анимации для первого экрана
  • В случае использование js lazy loading стоит убедиться в том, что не блокируется контент на первом экране сайта
