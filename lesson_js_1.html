Знакомство с JS

План

  1. Среда разработки
  2. Движки браузера

Где работает JS?

  • Любой браузер(Chrome, Opera, Safari, Firefox) на любой платформе (MacOS, Windows, Linux, Chromium, Android, iOS). Например, любой сайт(Facebook, Instagram, Google в браузер)
  • Прямо на компьютере(устройстве)(Node.js, Deno). Например, любой бекенд написанный на js(Paypal, Netflix)
  • На мобильных платформах(React Native). Примеры, мобильные приложения Facebook, Instagram

Где работает JS?

  • В виде десктопных приложений(Electron). Примеры приложений, Skype, Microsoft Teams, VS Code, Slack
  • Машинное обучение(tenzorflow)
  • Ещё много областей

Что может делать JavaScript в браузере?

  • Добавлять/менять html код, стили и тд
  • Реагировать на действие пользователя(клики, скроллы, перемещение курсора, нажатия клавиш)
  • Делать запросы на сервера, скачивать и отправлять файлы и прочая работа с сетью
  • Работать с куками(cookies), задавать вопросы пользователю, показывать ему сообщения
  • Сохранять на стороне клиента данные(local storage, session storage)

Что он не может?

  • Не имеет доступа к файловой системе устройства
  • Не может смотреть информацию и иметь доступ к другим вкладкам
  • Просто так нельзя общаться с серверами других доменов

Из чего состоит JavaScript в браузере

  • Сам движок javascript
  • DOM(Document Object Model): То что работает с DOM деревом(теги, элементы и работа с ними)
  • BOM(Browser Object Model): Работа с сетью, экраном, историей браузера и информация о нем

Что такое GitHub?

Репозиторий

Первые шаги

  1. установить git на компьютер;
  2. выполнить базовую настройку;
  3. инициализировать репозиторий;
  4. создать первый commit;
  5. зарегистрировать аккаунт на GitHub;
  6. залить код проекта на удаленный репозиторий.

Первоначальная настройка

      git config --global user.name "John Doe"
      git config --global user.email johndoe@example.com
      git config --global core.editor subl
настройки сохраняются в файле ~/.gitconfig
Первоначальная настройка Git

Базовые команды

Команда Описание
git status проверка состояния репозитория
git init инициализация нового репозитория
git add добавить файл(ы) в индекс для отслеживания
git commit сохранить изменения в репозиторий
git push актуализировать удаленный репозиторий
git pull актуализировать локальный репозиторий
git checkout переключение между ветками/коммитами
git clone создание локальной копии удаленного репозитория

.gitignore

Файл .gitignore содержит в себе пути к файлам/папкам, которые следует скрыть от системы контроля версий по тем или иным причинам.
